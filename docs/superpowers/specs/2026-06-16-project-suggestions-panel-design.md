# Project Suggestions Panel Design

## Objective

Add durable project suggestions that can be generated by autopilot or added manually, then surfaced inside each project page through a dedicated Suggestions panel.

## Problem

Today autopilot suggestions only exist in `AutopilotController` local component state and toast UI. They disappear on refresh, cannot be reviewed later, and have no workflow state.

## Chosen Approach

Create a dedicated Convex table named `projectSuggestions` and expose it through a project-level Suggestions panel in the project detail UI.

This table will store both:

- autopilot-generated suggestions
- manually added suggestions

This design was chosen over `activities` or embedding suggestions onto `projects` because suggestions are durable, stateful, and unbounded over time.

## Data Model

Add a `projectSuggestions` table in `convex/schema.ts` with these fields:

- `ownerId`: authenticated user who owns the suggestion record
- `projectId`: related project
- `source`: `autopilot | manual`
- `status`: `new | saved | dismissed | applied`
- `content`: suggestion body text
- `createdByUserId?`: optional attribution for manual suggestions if needed later

Recommended indexes:

- `by_ownerId_and_projectId`
- `by_ownerId_and_projectId_and_status`

## Backend API

Create a dedicated Convex module for project suggestions.

Public query:

- `listByProject(projectId)`
  - verifies ownership through the related project
  - returns a bounded, newest-first list for the selected project

Public mutations:

- `create`
  - inserts a suggestion for a project
  - validates that the current user owns the project
  - supports both `autopilot` and `manual` sources
- `updateStatus`
  - updates only the workflow state
  - validates ownership through the suggestion and project

No delete flow is required in the first version. `dismissed` covers the common hide/archive behavior without destructive operations.

## UI Design

Add a dedicated `ProjectSuggestionsPanel` component under `src/components/projects`.

The panel should appear inside the project detail page and include:

- a section title such as `Suggestions`
- a lightweight status filter or grouped tabs for `new`, `saved`, `dismissed`, and `applied`
- suggestion cards showing:
  - source badge: `Autopilot` or `Manual`
  - created timestamp
  - suggestion text
  - status actions

Suggested card actions:

- mark as `saved`
- mark as `applied`
- mark as `dismissed`
- return a suggestion to `new` if already triaged

Also include a small manual entry form in the same panel so users can create suggestions directly from the project page.

## Autopilot Integration

Update `AutopilotController` so generated suggestions are written to `projectSuggestions` in addition to being shown immediately in the UI.

Expected behavior:

- autopilot suggestion is generated
- suggestion is inserted with `source = autopilot`
- default workflow state is `new`
- existing inline feedback may remain for live visibility

The panel becomes the durable system of record for suggestion history.

## UX Rules

- `new` is the default state for every fresh suggestion
- `saved` means keep for later review
- `dismissed` means not useful right now
- `applied` means the user has acted on it, but it does not automatically create tasks or mutate projects in this version

The panel should remain useful even as suggestion volume grows, so the status filter is part of the initial contract rather than a later enhancement.

## Scope Boundaries

In scope:

- durable storage for project suggestions
- autopilot persistence
- manual suggestion creation
- status-based workflow
- project-page Suggestions panel

Out of scope for this version:

- automatic task or project creation from applied suggestions
- suggestion editing
- suggestion deletion
- global suggestions inbox across all projects
